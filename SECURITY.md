# Security Policy

## Supported Versions

We actively maintain security updates for the latest version of each product.

| Product | Supported |
|---|---|
| VibeMCP (latest) | Yes |
| Attestix (latest) | Yes |
| IssueVista (latest) | Yes |

## Reporting a Vulnerability

**Email:** info@vibetensor.com

**Subject line:** `[SECURITY] <brief description>`

**Please include:**
1. Description of the vulnerability
2. Steps to reproduce
3. Potential impact assessment
4. Suggested fix (if you have one)

### Response Timeline

- **Acknowledgment:** Within 24 hours
- **Initial assessment:** Within 72 hours
- **Status update:** Within 7 days
- **Resolution:** Based on severity (Critical: 48h, High: 7d, Medium: 14d, Low: 30d)

## Responsible Disclosure

- Do not publicly disclose the vulnerability before we have addressed it
- Do not access, modify, or delete data belonging to other users
- Do not perform denial-of-service attacks
- Provide sufficient detail for us to reproduce and fix the issue

We credit security researchers who report valid vulnerabilities (with permission) in our release notes.
